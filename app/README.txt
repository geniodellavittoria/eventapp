Punkte gemäss Anforderungen
 * 1 Punkt: Kommunikation über HTTP
 * 1 Punkt: Lokale Persistenz mittels Preferences oder Dateisystem
 * 2 Punkte: Verwendung von nicht im Module behandelten Bibliotheken / Frameworks
 * 1 Punkt: Verwendung von einfacher Nebenläufigkeit (ohne Services)
 * 1 Punkt: Implementierung & Verwendung eigene Server-Komponente

Features
 * Login
 * Registrierung
 * Übersicht Events
 * Event erstellen
 * Event bearbeiten
 * Event löschen
 * Events kategorisieren
 * An Event teilnehmen & abmelden
 * Manuellen Standort setzen für Eventsuche
 * Event filtern (inkl. nach Standort) und suchen

Technologien & Architektur
 * MVVM
 * Retrofit
 * Dagger 2
 * RxJava
 * Kotlin

Bemerkungen
Das Löschen der Events war in unseren Mockups nicht vorgesehen.
